Architectural Decision Records

Các quyết định kiến trúc quan trọng cùng lý do. Click để xem chi tiết.

ADR-001 DuckDB là scan engine chính — không load CSV vào pandas Accepted

Context

Prototype đầu dùng pandas để load CSV. File lớn gây OOM trên laptop 8GB RAM.

Decision

Tất cả profiling và quality checks chạy qua DuckDB SQL. DuckDB đọc thẳng từ CSV mà không load toàn bộ vào Python memory. Trade-off: phải viết SQL thay vì pandas, nhưng được: memory không giới hạn, quét nhanh hơn, evidence SQL có thể hiển thị cho người dùng.

ADR-002 Dual-Track LLM — cô lập Track kỹ thuật và Track nghiệp vụ Accepted

Context

LLM đóng vai "chuyên gia kinh doanh" bị cognitive bias: bỏ qua lỗi kỹ thuật nghiêm trọng để đi tìm insight doanh thu.

Decision

Hai luồng LLM độc lập hoàn toàn: Track 1 (per_table_insights) phân tích kỹ thuật trung lập. Track 2 (dual_track_overview) đoán domain từ schema rồi mới nhập vai chuyên gia. Hai output không bao giờ trộn lẫn nhau.

ADR-003 Web Runner là HTTP wrapper mỏng — không reimplementing pipeline Accepted

Context

Nguy cơ tạo hai code path song song: một cho CLI, một cho Web, dần diverge và khó bảo trì.

Decision

web_runner.py chỉ đơn giản gọi đúng hàm run_pipeline() từ cli.py. Web server xử lý file upload, lưu vào temp dir, truyền path cho pipeline. Frontend đọc artifact JSON qua API. Pipeline không biết gì về HTTP, web runner không có logic profiling.

ADR-004 Mermaid ER diagram với type sanitization bằng regex Accepted

Context

MermaidJS 11 từ chối attribute type chứa dấu phẩy (ví dụ: decimal(10,2)). Gây lỗi render silent trên database có decimal columns.

Decision

schema_diagram.py dùng re.sub(r'[^a-zA-Z0-9_]', '_', column.type) để loại bỏ mọi ký tự không phải identifier. decimal(10,2) → decimal_10_2. Trade-off readable vs luôn render được — chọn luôn render được.

Anti-Patterns — Tuyệt đối không implement lại

Những pattern đã được đánh giá và loại bỏ có chủ đích. Dev mới vào dự án PHẢI đọc phần này trước khi code.

❌ YAML Business Rules (cờ --rules)

Cờ --rules rules.yaml đã bị xóa. Người dùng phải tự viết YAML định nghĩa custom logic (ví dụ: date_shipped > date_ordered).

Tại sao bị loại bỏ: "Garbage in, garbage out." Bắt người dùng viết YAML là UX thất bại. Hệ thống đã có deterministic checks P0/P1 từ DBML schema (PK nulls, FK orphans, NOT NULL violations) mà không cần cấu hình. Thêm user-defined business rules trùng lặp những gì schema đã thể hiện và tạo gánh nặng bảo trì.
❌ LLM đơn luồng với role assignment sớm

Viết prompt bắt đầu bằng "Bạn là chuyên gia kinh doanh, hãy phân tích dữ liệu này..." trước khi trình bày lỗi.

Tại sao bị loại bỏ: Gây cognitive bias hallucination. LLM đóng vai "business expert" bỏ qua lỗi y tế để tìm doanh thu. Dual-Track Pattern (ADR-002) được implement để giải quyết vấn đề này. Role assignment chỉ xảy ra ở Track 2 sau khi Track 1 phân tích xong.
❌ pandas.read_csv() để profiling

Dùng pandas.read_csv() làm cách chính để scan CSV cho quality checks và thống kê.

Tại sao bị loại bỏ: Memory không giới hạn. File 500MB load vào pandas trên laptop 8GB RAM sẽ OOM. DuckDB đọc lazy từ disk với memory tối thiểu. Codebase có source scan guardrail trong benchmark report để phát hiện bất kỳ pandas.read_csv hay .fetchdf() không được guard.
❌ Python sinh HTML trực tiếp (html_merger pattern)

Build HTML bằng string concatenation hoặc thư viện Python thay vì Jinja2 templates.

Tại sao bị loại bỏ: Phá vỡ frontend/backend separation. Kiến trúc đúng là: Python sinh machine JSON → Jinja2 templates render HTML từ JSON đó. Web Dashboard đọc cùng JSON qua API. Trộn HTML generation vào Python business logic tạo monolith không testable.
❌ Probabilistic severity calibration

Dùng ML-based calibrator để "đoán" mức độ nghiêm trọng của null field.

Tại sao bị loại bỏ: DBML schema đã cho biết deterministic column nào là PK, FK, NOT NULL. Null ở PK luôn là P0, không có gì phải đoán. Thêm probabilistic ML lên trên knowledge deterministic này chỉ tạo nhiễu và unpredictability.
❌ Hosted/remote Python job runner

Deploy profiler như cloud service, người dùng upload data lên server từ xa.

Tại sao bị loại bỏ: Đây là explicit non-goal. Tool là local-first theo thiết kế — data không bao giờ rời khỏi máy người dùng. Web runner bind tới 127.0.0.1 only. Đây là security requirement, không phải technical limitation.
